# Security Policy

## Supported Versions

We only provide security updates for the latest version of the repository.

| Version | Supported          |
| ------- | ------------------ |
| Main    | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability within this project, please report it via GitHub Issues or contact the maintainer directly.

### Our Process
1.  **Report**: Open an issue with "[SECURITY]" in the title.
2.  **Verify**: We will verify the vulnerability within 48 hours.
3.  **Patch**: A patch will be developed and merged.
4.  **Disclose**: Once the patch is live, we will update the community.

Please do not disclose security vulnerabilities publicly until they have been addressed.
